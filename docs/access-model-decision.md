# Gwynne Park Run Club - Access Model Decision

Decision date: 2026-06-10

This is the locked-in access model for the live signup/login system.

## Staff And Coaches

- Staff accounts are invite-only.
- Coaches cannot create public self-serve accounts.
- Owners/admins invite staff by email.
- Invited staff use Supabase Auth with password or magic-link login.
- Staff roles are school-scoped: owner, admin, coach, and optional kiosk helper.
- Kiosk access remains behind an authenticated staff/admin session.

## Students And Athletes

- Students do not need passwords.
- Student access is passwordless through barcode, QR code, or a simple private access code.
- Student access must be read-only except approved student-facing actions such as viewing goals, awards, training, and optional non-scoring reflections.
- Do not use a predictable username like first initial plus last name as the only access key.
- A friendly handle such as `JSmith` can be displayed or used as a helper label, but the actual access credential must include a non-guessable token.
- Example safe codes: `JSMI-7KQ`, `JSMITH-482`, or an opaque QR/barcode token generated by the backend.

## Parents And Guardians

- Parents can search for their child by name, but search results must be privacy-limited.
- Before a full child profile opens, the parent must confirm with a school-generated guardian code/link.
- Search results should show partial identity only, such as first name plus last initial, year group, and class if needed.
- Guardian links are child-specific, revocable, expiring, and logged.
- Parent access is read-only and limited to the linked child profile, awards/certificates, goals, training visibility, and progress summaries.

## Launch Rules

- Universal `DEMO` access must be disabled before real student data is entered.
- Every staff, parent, and student access path must be school-scoped.
- Student and parent access tokens must be non-guessable and generated by the backend.
- Access events should be audit logged, especially guardian link creation, guardian login, staff invites, role changes, kiosk use, and exports.
