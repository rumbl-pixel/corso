# Corso Beta Tester Checklist

Use demo data only. Do not enter real student names, medical details, parent details, or school credentials.

## Quick Test Path

1. Open the home page.
   - Confirm the Corso logo/name appears.
   - Confirm the beta demo warning is visible.
   - Confirm About Us and Privacy Policy links open.

2. Log in to the admin demo.
   - Confirm the admin area opens.
   - Confirm the Admin safety warning is visible.
   - Confirm the main tab buttons fit on laptop and phone widths.

3. View students.
   - Search for a demo student.
   - Open a student profile from the Students list.
   - Confirm admin mode stays active.

4. Check scanner/kiosk.
   - Start a demo scanner session.
   - Test typing/scanning a demo barcode.
   - Confirm duplicate scan protection.
   - Open kiosk and confirm it is staff/admin controlled.

5. Open Sports.
   - Turn on Interschool Athletics Mode.
   - Open Team Checklist.
   - Open an event team popup.
   - Confirm Junior/Intermediate/Senior filters show the correct year groups.
   - Save a demo team and review Team List Summary.

6. Open Programming.
   - Drag or select lesson items into Starters, Sessions, and Cool-downs.
   - Change minutes.
   - Open Mini Coach and ask for a local planning suggestion.

7. Check mobile/iPad layout.
   - Home, Admin, Student Profile, Sports, Programming, and Parent should not horizontally clip.
   - Modals should open centred and stay usable.
   - Buttons and pills should remain readable.

8. Report confusing parts.
   - Use the Feature Suggestion / Send Feedback button.
   - Include the page name, device, and what felt confusing.

## What To Watch For

- Text that is too light in dark mode.
- Buttons that look clickable but do nothing.
- Information cards that accidentally highlight like buttons.
- Popups opening off-centre.
- Any place that appears to invite real student data during beta.
- Any workflow where Sports/Interschool attendance changes Run Club lap totals.
